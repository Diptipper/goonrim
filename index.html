Preface

Skyrim is an old game. But despite its dated mechanics and buggy reputation, it has one of the most active fanbases amongst all single player games. I'd argue it's because of the modding scene, R18 or not.

When Skyrim was launched in 2011, Bethesda was kind enough to release the Creation Kit for modders to use. This lets us modders explore the inner working of the Creation Engine and create many incredible mods out of it. But versatility of the tool comes with its technicality. 1. How modding works

Skyrim as a game is composed of two parts: the plugins and the assets. The plugins handle records of stuff inside the game (basically the blueprints). At runtime, the engine reads the plugins and pulls the assets associated with the record and realizes it as stuffs inside the game.

Both the plugins and the assets are stored inside the Data folder of your Skyrim directory. For clarity, when I say root folder, I'm talking about the folder that contains the game binary (SkyrimSE.exe). And when I say the data folder, I'm talking about literally the 'Data' folder inside the root directory.

Plugins are contained in files with the following extensions: .esm (Elder Scrolls Master), .esp (Elder Scrolls Plugin), .esl (Elder Scrolls Light). The rest of the stuff inside the Data folder are the assets.

Many mods require you to install external frameworks for extra functionalities. These are typically installed in the root folder. I'll mention about this more below.

1.1 Handling mod conflicts

When you install mods, both plugins and assets can have conflicts with other mods. It is your responsibility to manage these conflicts. Otherwise, the game will start to act weird, if not crash to desktop entirely.

Plugin conflicts
This is the easy one. If the conflict is just on the plugin level. You can manage the conflicts by adjusting the load order. Mods loaded after/below are prioritized over mods loaded before/above. You can adjust the load order before launching the game, but honestly, don't do that. I recommend using a proper mod manager such as Mod Organizer 2 or Vortex. I am using Vortex, so the screenshots will be from Vortex. In these mod managers, there is an option to auto-sort the load order to minimize obvious dependency errors. But you have to decide the less-obvious conflicts by yourself again.

Asset conflicts
This one is tricky. To explain why, remember that we are talking about files here. When two mods use their own assets; e.g., skin textures, you cannot keep both of them and decide which one to use later. Once you overwrite the files, it's permanent. That's why using mod manager is so important since they handle these kind of things for you.
